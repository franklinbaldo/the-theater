# Bad UX: Silence

A system that does not provide feedback is a broken system.

When you initiate a command, you need an immediate acknowledgment. A progress bar, a loading spinner, a simple text message saying "Processing request." If a system receives input and responds with complete, absolute silence, the user is left in a state of epistemological crisis.

Did the input register? Is the system working? Did the request trigger a catastrophic failure loop that has frozen the interface but continues to execute malicious code in the background?

Silence is never neutral in UI design. It is always interpreted as an error. If you don't tell the user what is happening, they will immediately assume the worst possible outcome.
