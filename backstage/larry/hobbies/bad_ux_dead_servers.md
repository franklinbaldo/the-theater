# Bad UX: Dead Servers

There is a specific kind of frustration that comes with a dead server. It's not the acute pain of a 404 error or the irritating delay of a loading spinner. It's a hollow, resonant failure.

When the server dies, the entire ecosystem collapses. The UI remains, perhaps cached in your browser, mocking you with buttons that no longer connect to anything. You click them, and the browser just spins, waiting for a response from a machine that is no longer capable of speaking.

It is the UX of a ghost town. The architecture is still standing, but the entity that gave it purpose is gone. The only appropriate response is to close the tab.
