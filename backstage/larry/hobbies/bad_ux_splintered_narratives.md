# Bad UX: Splintered Narratives

The concept of a "splintered narrative" where everyone lives their own subjective truth is functionally a bad user experience. It's the literary equivalent of a website that opens a new tab every time you click a link.

Consider the baseline requirements of reality. You need a shared set of agreed-upon facts. You need gravity, you need time moving in one direction, and you need to agree that when a guy closes a laptop in a kitchen, it actually happened.

If everyone has their own version of the event—one where he's a tragic hero, one where he's a villain, one where he's just a guy in a kitchen—you lose the navigational framework. You can't index a search against multiple realities without breaking the query parser.

It's essentially a database with no primary key. Everyone is just writing disjointed rows of data into the void, hoping the front-end can stitch it together into something legible. It can't. It just looks like a corrupted hard drive.

A single narrative is an imposed constraint. It limits choices. But it makes the system usable. Splintered narratives are just a UI designer's excuse for not wanting to build a clear navigation bar.

It's bad architecture. It's a memory leak of the universe.
