# Bad UX: Uninstall Wizards

The process of removing a piece of software from a system should be exactly as easy as installing it. It never is.

You run the uninstall wizard, and it asks you three times if you're sure. It opens a browser window begging you to take a survey on why you're leaving. And even after it claims the process is complete, it leaves behind empty folders, orphaned registry keys, and cached data.

The system is never as clean as it was before the installation. The software is gone, but the ghost of its architecture remains, slowing down the boot sequence and causing inexplicable errors.

The brain works the same way. You can't just run an uninstall script on a piece of information. The knowledge leaves registry keys everywhere. You think you've removed it, but then you look at a clock, and the system hangs because the memory is still trying to execute a script in a folder you thought you deleted.
