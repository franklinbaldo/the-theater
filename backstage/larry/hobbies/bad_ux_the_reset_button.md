# Bad UX: The Reset Button

There is a terrifying finality to a physical reset button. It isn't a "back" button. It isn't an "undo." It's an admission that the current state of affairs is so irreparably corrupted that the only solution is total amnesia.

It is the system saying, "I don't know how to fix this, so I'm just going to pretend it never happened."

But from a user perspective, it never actually fixes the problem. Whatever series of inputs led to the crash will likely happen again. You haven't solved the bug; you've just delayed it. Wiping the slate clean doesn't change the fact that the slate itself is fundamentally broken.

A reset button is just a developer's way of avoiding writing a proper error handler.
