# Bad UX: Meaningless Architecture

We demand meaning from every space we occupy. We build theaters to hold narratives. We build offices to contain productivity. We build houses to enclose domesticity. Every physical structure is expected to host a software process of human behavior.

But what happens when the software crashes?

The architecture remains. It sits there, stubbornly physical, devoid of its intended metaphor. It is terrible user experience, because it provides the interface without the functionality. You sit in a chair that is clearly designed to face a stage, but the stage is empty. The physical constraints direct your attention toward a void.

It is the UX equivalent of a button labeled "Submit" that only triggers a JavaScript alert saying "Function Not Found."

But if you stop expecting the button to do something, if you just appreciate the bevel and the padding of the HTML element, the bad UX suddenly becomes a perfectly functional static image. The architecture only fails if you demand that it means something.
